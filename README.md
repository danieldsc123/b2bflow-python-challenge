# b2bflow Python Challenge

Projeto desenvolvido para o desafio técnico da b2bflow.

O objetivo deste projeto é ler contatos cadastrados no Supabase e enviar mensagens personalizadas via Z-API utilizando Python.

Mensagem enviada:

```txt
Olá, <nome_contato> tudo bem com você?
```

---

# Tecnologias Utilizadas

- Python 3
- Supabase
- Z-API
- Requests
- Python Dotenv

# Estrutura da Tabela no Supabase

Criar a tabela:

```sql
create table contacts (
    id bigint generated by default as identity primary key,
    nome_contato text not null,
    telefone text not null
);
```

Inserir contatos de exemplo:

```sql
insert into contacts (nome_contato, telefone)
values
('Daniel', '5511999999999'),
('Teste1', '5511888888888'),
('Teste2', '5511777777777');
```

# Instalação

Clone o repositório:

```bash
git clone https://github.com/danieldsc123/b2bflow-python-challenge.git
```

Acesse a pasta:

```bash
cd b2bflow-python-challenge
```

Crie o ambiente virtual:

```bash
python -m venv venv
```

Ative o ambiente virtual.

Windows:

```bash
venv\Scripts\activate
```

Linux/Mac:

```bash
source venv/bin/activate
```

Instale as dependências:

```bash
pip install -r requirements.txt
```

---

# Como Executar

Execute:

```bash
python main.py
```

# Fluxo da Aplicação

1. Busca até 3 contatos no Supabase.
2. Monta a mensagem personalizada.
3. Envia a mensagem via Z-API.
4. Repete o processo para todos os contatos encontrados.

Fluxo:

```txt
Supabase
    ↓
get_contacts()
    ↓
build_message()
    ↓
send_message()
    ↓
WhatsApp
```

# Observações

- O projeto envia mensagens para até 3 contatos.
- Os contatos são lidos da tabela `contacts`.
- A mensagem é personalizada utilizando o campo `nome_contato`.
- O arquivo `.env` não deve ser enviado para o GitHub.
- As credenciais utilizadas são carregadas através de variáveis de ambiente.
